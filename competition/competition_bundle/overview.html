Overview

The Competition

Autonomous driving (AD) is the next frontier of artificial intelligence and machine learning. Intending to further research in AD, we invite you to participate in an autonomous driving competition organised by Huawei Technologies Ltd. and academic collaborators.

Tasks and Evaluation

This competition seeks to advance autonomous driving by developing agents that can drive as quickly and safely as possible from the start to destination amid background traffic. Data for the competition are mined from large-scale naturalistic driving data, manipulated, and replayed in SMARTS simulation environment. The following typical driving scenarios are tested: cruising, overtaking, merging, left turns at unsignalized intersections and being cut off by another vehicle. For some scenarios, interactive background vehicles are added in SMARTS.

Agents will be ranked according to their task completion (i.e., % of completed scenarios), completion time (e.g., time steps taken to complete scenario), human-likeness (e.g., jerkiness, acceleration, and distance to surrounding vehicles, etc), and safety (e.g., traffic rule violations).

Competition Tracks

There are two tracks in the competition.

Track 1: The participants may use any method to develop their solutions.

Track 2: The participants are only allowed to train their models on offline datasets.

Winners in each track will receive cash prizes and will get a chance to present their innovative solutions during the virtual ceremony.

Prizes

Top participants in each track will receive the following prizes:
  • Gold US$6000
  • Silver US$4000
  • Bronze US$2000
Additional prizes:
  • US$1000 for the most innovative approach out of top-6 entries in both tracks.
  • US$1000 given to one of the valid submissions not in top-3 positions in either track.

Code

(a) Use python3.8 to develop your model.
(b) All necessary code for the competition is provided at SMARTS/competition/ directory in branch comp-1 of the SMARTS repository.
(c) Install SMARTS package from comp-1 branch as:
$ pip install "smarts[camera-obs] @ git+https://github.com/huawei-noah/SMARTS.git@comp-1"
(d) To obtain the base code, run the following:
$ git clone https://github.com/huawei-noah/SMARTS.git
$ git checkout comp-1
$ cd <path>/SMARTS/competition

Extended Environment "bubble_env"

(a) An additional evaluation environment is available and it is used as part of the evaluation.
(b) It can be acquired in a few ways. The easiest way is to install it through pip.
$ pip install "bubble_env @ git+https://malban@bitbucket.org/malban/bubble_env.git"
(c) To directly obtain the additional environment and resources you may instead do the following:
$ cd <path>
$ git clone https://malban@bitbucket.org/malban/bubble_env.git
$ cd bubble_env
$ git lfs pull # or follow instructions in ./README.md
$ pip install -e .
$ cd <path>/SMARTS/competition

Validation

(a) Since Track-1 and Track-2 has limited number of submissions, a Validation stage is made available.
(b) Participants may submit their zipped Track-1 and Track-2 code to Validation stage, to check whether their code executes without errors.
(c) Scores will not be revealed in this stage.

Track-1

(a) Objective is to train a single policy capable of controlling single-agent or multi-agent to complete different driving tasks in various scenarios.
(b) Any method such as reinforcement learning, offline reinforcement learning, behaviour cloning, generative models, predictive models, etc, may be used to develop the policy.
(b) For more instructions and explanations, see the accompanying README.md file of each stage.
  • Training: /SMARTS/competition/track1/train/README.md
  • Submission: /SMARTS/competition/track1/submission/README.md
  • Evaluation: /SMARTS/competition/evaluation/README.md

Track-2: Offline Learning

(a) Objective is to train a single policy capable of controlling single-agent or multi-agent to complete different driving tasks in various scenarios.
(b) An offline learning method using offline datasets must be used to develop the policy.
(c) For more instructions and explanations, see the accompanying README.md file of each stage.
  • Training & Submission: /SMARTS/competition/track2/README.md
  • Evaluation: /SMARTS/competition/evaluation/README.md
